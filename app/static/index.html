📐 Strain Map Agent

Measures local lattice distortions by comparing atom-pair distances to the median reference spacing.

ℹ️ WHAT THIS AGENT DOES

This agent computes local row and column strain across the entire field of view. It takes the detected atom positions from the Peak Finding step, pairs each atom with its nearest neighbours, and compares those spacings to the global median (the "unstrained" reference). The result is a series of colour-coded strain maps — one for each direction — highlighting regions of compression or tension in the crystal lattice.

INPUTS

The _atoms.csv file produced by the Peak Finding Agent. Find it in your project folder under 2_peak_finding/. It contains x, y coordinates and intensity for every detected atom.

Your original raw image from the microscope — the exact same file you uploaded to the Peak Finding Agent. Used as the background for the overlay maps.

STRAIN TUNING

Type: float. UI range: min 0.0, no hard max. Recommended start: 0.0 to 0.2. Use 0 to disable pruning. Increase only when very close dumbbell pairs are corrupting the lattice basis.

Type: float. UI range: 0.2 to 2.0. Recommended practical range: 0.7 to 1.7. Higher values accept more neighbors as belonging to the same row or column and usually increase strain coverage.

Type: integer. UI range: 0 to 8. Recommended practical range: 1 to 2. `0` maximizes coverage but is usually too permissive for scientific use. Lower values increase atom retention; higher values are stricter and may leave many atoms without strain.

🧩 Clustering Agent

Groups atoms into distinct structural environments using machine learning (HDBSCAN + GMM).

ℹ️ WHAT THIS AGENT DOES

This agent uses unsupervised machine learning to group atoms that share similar local structural environments (strain, intensity, neighbour distances). It first applies HDBSCAN (a density-based clustering algorithm) to find natural groupings without needing to specify the number of clusters. Then it fits a Gaussian Mixture Model (GMM) to refine the boundaries. The output is a colour-coded map where each colour represents a distinct structural environment — e.g. interfacial atoms, bulk atoms, defect-rich zones.

INPUTS

The *_strain.csv output from the Strain Map Agent. Find it in your project folder under 3_strain_map/. It contains per-atom strain values used as features for clustering.

Drop *_strain.csv here

Your original raw STEM image from the microscope — the same file used in Peak Finding. If provided, atom clusters are drawn overlaid on the real image for visual inspection.

Drop the original raw STEM image here (optional)
CLUSTERING PARAMETERS
The minimum number of atoms to form a cluster. Start with 15. Increase (e.g. 30) to find fewer, larger groups; decrease (e.g. 5) to detect small isolated clusters.
Number of structural environments you expect in your sample. Start with 4. For a simple crystal: 2–3. For a heterostructure interface: 4–6.

🔴 Defect Region Screening Agent

Builds a smooth abnormal-region heatmap by fusing peak-finding irregularity, strain distortion, and clustering abnormality.

ℹ️ WHAT THIS AGENT DOES

This stage replaces isolated atom-defect counting with region screening.
It combines three existing signals from the pipeline:
• Peak-finding irregularity: where dumbbell pairing or sublattice structure breaks down.
• Clustering abnormality: where local atomic environments differ from the dominant lattice.
• Strain distortion: where the measurable distortion field is strongest.
The result is a heatmap and region overlay that highlight abnormal zones rather than a small list of isolated atom defects.

📁 INPUT GUIDE

This agent requires the Clustered CSV (filename ends with _clustered.csv).
You can find it in the output folder of the Clustering Agent.
